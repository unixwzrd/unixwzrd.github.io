Introducing Secrets Kit v0.9.0.

Secrets Kit is a local macOS command-line tool for handling API keys, tokens, passwords, PII, and other sensitive values without leaving them scattered across `.env` files, shell startup files, copied commands, and project directories.

It stores secret values in the macOS login Keychain, keeps non-secret metadata in a local registry, and exports environment variables only when a local runtime actually needs them.

This is not a vault service and it is not a claim of perfect security. It is a practical step toward better local secret hygiene and a better alternative to plain-text secret sprawl.

That makes it useful for agent runtimes, local UIs, shell workflows, development stacks, and other environments that still rely on environment variables but should not rely on secrets living all over the filesystem.

Title:
Introducing Secrets Kit

Subtitle:
A local macOS tool for keeping tokens, passwords, API keys, and other sensitive values out of scattered plain-text files.

Body:
Secrets Kit is a new local macOS command-line tool built around a very practical problem: too many sensitive values end up spread across `.env` files, shell startup files, copied commands, and project directories.

Instead of leaving those values in plain text, Secrets Kit stores secret values in the macOS login Keychain, keeps non-secret metadata in a local registry, and exports environment variables only when the current shell or runtime actually needs them.

This is not a hosted secret manager or a promise of perfect security. It is a pragmatic tool for improving local secret hygiene and reducing the kinds of mistakes that lead to accidental GitHub commits or other avoidable exposure.

That makes it a good fit for local developer workflows, agent runtimes, shell automation, and any stack that still needs environment variables but should not depend on raw secrets living around the filesystem.
