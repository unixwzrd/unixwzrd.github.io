I just released **LLM-Ops-Kit**.

It is an operational toolkit for running, deploying, debugging, and maintaining self-hosted AI stacks across hosts. The project started around a local OpenClaw deployment, but the tooling is useful anywhere you need reproducible wrappers, model profiles, bridge services, runtime maintenance, streamlined Jinja/ChatML with tool calling, and better observability for local AI systems.

This one came out of real-world operational pressure: stale runtime state, mismatched ports, prompt-template debugging, TTS bridge routing, cross-host deployment, OpenAI-compatibility problems, and the kind of failures you only find during real end-to-end testing.

Just released this, it was bit of work, mostly to get OpenClaw to run with minimal resources. Several highlights are:

- An efficient Jinja Template for ChatML trained models, it eliminates redundant information, like thoughts, tool calls large graphics which pollute the context window (n_ctx). A final filter for your prompt.
- Native MLX TTS with voice cloning using MLX-Audio, and a patch in my repo which aligned with the OpenAI API and ensures that the sample audio and text transcript are applies correctly.
- All local models, SAVES $$$ on API calls and also is completely private.
- Includes framework for Embedding models
- Uses the Qwen3.5-35B-A3B from #HuggingFace as its core model.

Body:
LLM-Ops-Kit is now live for macOS.

It is the operator layer for a self-hosted AI stack: service wrappers, model profiles, bridge services, deployment helpers, runtime maintenance, streamlined Jinja/ChatML template variants, and the tooling needed to debug what the stack is actually doing.

The project started around a local OpenClaw deployment, but the scope is broader than that. If you are managing models across hosts, dealing with local bridges and remote inference servers, and trying to make the whole system reproducible instead of fragile, this project is for that class of work.

A big part of the payoff so far has been local TTS and voice cloning, which cuts costs and keeps more of the stack under your own control.
